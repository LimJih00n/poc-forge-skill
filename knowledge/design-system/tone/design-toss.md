# Toss Media Reference

이 문서는 OpenAI 생성 UX 화면 이미지의 시각 참고용 메모입니다. Dealcatch 자동 생성 포맷에서는 `04-ux/assets/provided-media/images/`의 생성 이미지를 proposal page에 직접 표시하며, 이미지가 없으면 제공 예정 상태로 둡니다.

## 사용 방식

- Toss 계열의 정보 밀도, 여백, 카드/리스트/CTA 비례, 신뢰감 있는 문구 톤을 이미지 프롬프트의 참고 요소로 사용합니다.
- 실제 proposal page에는 `04-ux/assets/provided-media/images/`의 생성 화면 이미지만 표시합니다.
- 참조 브랜드명과 내부 코드명은 최종 proposal page 문구에 노출하지 않습니다.

## 참고 포인트

- 금융/결제/업무 흐름은 높은 대비보다 명확한 정보 계층, 상태 구분, 안정적인 CTA 배치를 우선합니다.
- 모바일 우선 화면은 첫 viewport의 정보량, 하단 CTA, 카드 높이와 여백을 구체적으로 지정합니다.
- Pretendard 계열 sans-serif를 기본으로 보고, `font-weight`는 700 이하로 제한합니다.
- 여러 화면 이미지가 서로 다른 앱처럼 보이면 실패로 보고 팔레트와 컴포넌트 비례를 먼저 맞춥니다.
